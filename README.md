# 📋 업무현황판 (Worklog Management Dashboard)

> TYM 중앙기술연구소의 **일일 업무일지 기록 + 프로젝트 진행 추적**을 하나로 묶은
> Microsoft Power Apps 기반 사내 업무 관리 시스템

연구원 개개인이 매일의 업무를 손쉽게 기록하고, 팀장·부문장·소장은 조직의 업무 현황을
역할에 맞게 한눈에 들여다볼 수 있도록 설계되었습니다. 약 **103명**이 매일 사용하는
실서비스이며, Microsoft Teams 안에서 탭으로 바로 열립니다.

---

## ✨ 한눈에 보기

|------|------|
| **무엇** | 일일 업무일지 작성 + 프로젝트/업무 진행 현황 관리 |

## 🎯 이 시스템이 풀어주는 문제

기존에는 업무일지를 엑셀이나 수기로 관리해 다음과 같은 어려움이 있었습니다.

- 매일 같은 정보를 반복 입력해야 함
- 팀·프로젝트별로 누가 어떤 일을 하는지 모으기 어려움
- 업무 시간·작업 내용이 흩어져 집계가 번거로움

**업무현황판**은 프로젝트 → 개인 업무 → 일일 기록을 하나의 흐름으로 연결해,
한 번 등록한 업무를 매일 골라 기록만 하면 자동으로 집계·시각화되도록 만들었습니다.

---

## 🗂️ 데이터가 흐르는 구조

이 시스템은 세 단계의 데이터가 위에서 아래로 연결됩니다.

```
┌─────────────────────────────────────────────┐
│  Project_DB  (프로젝트 마스터)                  │
│  회사가 관리하는 전체 프로젝트 목록               │
└───────────────────┬─────────────────────────┘
                    │  내가 할 프로젝트를 고르면
                    ▼
┌─────────────────────────────────────────────┐
│  Task_DB  (개인 업무 등록)                      │
│  "이 프로젝트로 이런 업무를 한다" 등록            │
└───────────────────┬─────────────────────────┘
                    │  매일 그날 한 일을 고르면
                    ▼
┌─────────────────────────────────────────────┐
│  Daily_DB  (일일 업무일지)                      │
│  날짜별 작업 내용 · 작업 시간 · 작업 분류 기록     │
└─────────────────────────────────────────────┘

         + UserInfo_DB (사용자 정보 · 팀 · 직급 · 권한)
```

| 리스트 | 역할 |
|--------|------|
| **Project_DB** | 회사가 관리하는 프로젝트 마스터 (업무_계정·PM·기간·상태 등) |
| **Task_DB** | 사용자가 프로젝트를 골라 등록한 "내 업무" |
| **Daily_DB** | 매일 작성하는 업무일지 (작업 내용·시간·분류) |
| **UserInfo_DB** | 이름·이메일·팀·부문·직급 등 사용자 정보 |

## 🖥️ 주요 화면

### 📝 업무일지 (Daily_Screen)
매일의 업무를 기록하는 메인 화면입니다.
- 왼쪽 달력에서 날짜 선택 (작성한 날은 표시됨)
- 등록해 둔 업무 중 그날 한 일을 골라 **작업 시간·작업 내용** 입력
- 업무마다 **작업 분류**(회의·문서/보고·개발 등)를 선택해 저장
- 연차·반차 선택 시 작업 시간(8h·4h) 자동 입력
- 같은 업무를 여러 날에 한 번에 기록하는 기간 입력 지원

### 📌 업무현황 (Task_Screen)
내 업무를 칸반보드로 관리합니다.
- **시작전 → 진행중 → 완료** 3단계로 한눈에 파악
- 왼쪽에서 프로젝트를 체크/해제해 보드 필터링
- 프로젝트를 골라 새 업무 등록
- **프로젝트 마스터 관리 모달** — 업무 계정·상태별 필터, 조회/신규/수정/삭제,
  D-day·진행률·등록일을 보여주는 2단 상세 화면

### 📅 업무조회 (View_Screen)
조직의 주간 업무를 매트릭스로 조회합니다.
- **팀별 / 프로젝트별** 두 가지 보기 모드
- 사람(행) × 요일(월~금, 열)로 한 주의 업무를 한눈에
- 이전 주 / 다음 주 / 이번 주로 자유롭게 이동
- 연차·반차는 색으로 구분 표시
- 업무에 대한 **피드백** 작성 가능
- **역할에 따라 보이는 범위가 다름** (아래 권한 참고)

### 📊 대시보드 (DashB_Screen)
업무 데이터를 시각화합니다.
- 업무 계정 / 프로젝트 / 작업 분류별 **분포 차트**
- 총 작업 시간, 1인 평균, 진행중 업무 수 등 **KPI**
- 기간(이번 주·이번 달·사용자 지정) 필터
- 전 기간 대비 주요 변화 Top3 분석

## 🔐 권한 체계

SharePoint 자체 권한이 아니라 **앱 내부 로직**으로 역할별 조회 범위를 제어합니다.

| 역할 | 조회 범위 |
|------|-----------|
| **팀원** | 본인 팀이 작성한 업무일지 |
| **팀장** | 본인 팀이 작성한 업무일지 |
| **부문장** | 담당 부문(Function / Project) 팀장들의 업무일지 |
| **소장** | 전체 팀장들의 업무일지 |

## 🛠️ 기술 스택

| 영역 | 사용 기술 |
|------|-----------|
| 앱 | Microsoft Power Apps (Canvas App) |
| 수식 언어 | Power Fx |
| 데이터 | SharePoint Lists |
| 배포 | Microsoft Teams 탭 |
| 메일·알림 | Office 365 Outlook (표준 커넥터) |

> 무료 Power Platform 라이선스 환경에서 운영됩니다.
> Dataverse·프리미엄 커넥터를 사용하지 않고 SharePoint Lists만으로 구현했습니다.

### ⚙️ 성능 설계 메모
대용량 테이블(Daily_DB, 연 4.5만 건+)은 PowerApps 위임 한도(2,000건)에 걸리기 쉽습니다.
이 앱은 다음 원칙으로 체감 속도와 정확도를 함께 잡았습니다.

- **주간 조회는 날짜 범위 필터로 Daily_DB를 직접 호출** (날짜 필터는 위임 가능)
- 자주 쓰는 사용자·프로젝트 정보는 앱 시작 시 **Collection에 캐싱**
- `Distinct`·`in` 등 위임 불가 함수를 피하고, 단일 값 비교 위주로 필터 구성
- 프로젝트 선택 필터는 **"해제된 항목만 추적"** 방식으로 위임 한계를 우회

## 📌 개발 현황

지속적으로 버그를 수정하고 기능을 추가하며 운영 중입니다. 진행 상황과 상세 기술 명세는
[`CLAUDE.md`](./CLAUDE.md)에서 관리합니다.

**최근 작업**
- 프로젝트 관리 모달 2단 레이아웃 재설계 (조회/신규/수정/삭제, D-day·진행률·등록일, 상태 뱃지·필터)
- 업무조회 주 이동 시 매트릭스 정상 갱신
- 프로젝트 선택 필터 정확도 개선 (해제 항목 추적 방식으로 누락 해결)
- 연차·반차 작업 시간 자동 입력
- 작업 분류 선택 시 입력값이 사라지던 문제 해결
- 사용자 이름 인식 로직 안정화 (이메일·사용자 정보 기준으로 통일)
- 팀원도 팀 전체 업무를 조회하도록 권한 범위 조정

**예정 작업**
- 연장근로·출장·연차·공지 시스템 추가
- 업무조회 매트릭스 재계산 로직 단일화 (유지보수성 개선)
- 연차·반차 색 구분 등 시각화 개선
- 대용량 데이터 조회 성능 최적화
